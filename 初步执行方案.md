# Steam 自动下载安装加速器 —— 详细实现指南（AI就绪版）

## 📌 项目目标

开发一个 **Windows GUI 工具**，实现：

1. **自动下载**：从官方源获取 `SteamSetup.exe`
2. **静默安装**：无人值守安装 Steam 客户端
3. **网络加速**：通过 Hosts 优化或本地代理提升下载速度

## 1. 总体流程（三阶段）

| 阶段 | 动作 | 说明 |
|------|------|------|
| **准备** | 权限检查、磁盘空间验证、网络检测 | 确保环境可执行 |
| **加速 + 下载** | 修改 Hosts（或启动代理）→ 下载 SteamSetup.exe | 加速是关键亮点 |
| **安装 + 清理** | 静默安装 → 监控进程 → 验证结果 → 恢复Hosts（可选） | 无人值守完成 |

### 详细执行步骤

#### 阶段一：准备
1. **检查管理员权限**
   - 使用 `System.Security.Principal.WindowsPrincipal` 判断当前进程是否以管理员身份运行。
   - 若无权限，提示用户“需要管理员权限”，并通过 `Process.Start` 重新以管理员身份启动自身（`runas` 动词）。
2. **检查磁盘空间**
   - 获取目标安装分区（默认系统盘或用户指定盘）的可用空间。
   - 至少需要 **2 GB** 空闲空间（Steam本身约1.5GB + 临时文件）。
   - 若空间不足，提示用户清理空间或更换安装目录。
3. **网络连通性检测**
   - 尝试向 `https://steamcdn-a.akamaihd.net` 或 `https://store.steampowered.com` 发送 HTTP HEAD 请求，超时5秒。
   - 若不通，提示用户检查网络/防火墙。
4. **检查已有 Steam 进程**
   - `Process.GetProcessesByName("steam")` 查找是否已有 Steam 运行。
   - 若有，弹出选择框：“检测到Steam正在运行，是否自动关闭并继续？” 若同意则发送 `taskkill /f /im steam.exe`。
5. **备份重要状态**
   - 记录当前时间戳，生成会话日志文件。
   - 读取并备份原始 Hosts 文件到临时目录（用于恢复）。

#### 阶段二：加速 + 下载
1. **获取加速 IP 列表**
   - 从内置资源或远程 API 获取最新的 Steam 相关域名 → IP 映射表。
   - 至少包含：`steamcommunity.com`, `store.steampowered.com`, `cdn.steamstatic.com`, `steamcdn-a.akamaihd.net` 等。
2. **写入 Hosts 加速**
   - 将获取到的域名-IP 追加到 `C:\Windows\System32\drivers\etc\hosts` 末尾。
   - 确保不重复写入。
   - 刷新 DNS 缓存：执行 `ipconfig /flushdns`。
3. **下载 SteamSetup.exe**
   - 从官方 URL `https://cdn.cloudflare.steamstatic.com/client/installer/SteamSetup.exe` 下载。
   - 支持断点续传：若临时文件已存在，读取已下载大小，设置 HTTP `Range` 头从断点续传。
   - 显示下载进度（总大小、已下载、速度）。
   - 校验文件大小是否与服务器 Content-Length 一致。

#### 阶段三：安装 + 清理
1. **静默安装**
   - 构造参数 `"/S"` （必须）；若用户指定目录，追加 `" /D=路径"`（注意：路径不加引号，且该参数必须最后）。
   - 启动安装进程，不显示窗口。
   - 等待安装进程退出（或监控 Steam.exe 出现）。
2. **验证安装**
   - 检查注册表键值 `HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Valve\Steam\InstallPath` 是否存在且指向有效路径。
   - 确认 `steam.exe` 存在于该目录。
3. **可选恢复 Hosts**
   - 根据用户设置，将备份的 Hosts 文件复制回原位置。
   - 再次执行 `ipconfig /flushdns`。
4. **清理临时文件**
   - 删除下载的 `SteamSetup.exe`。
   - 删除 Hosts 备份文件（如不再需要）。
5. **完成通知**
   - 界面显示“安装成功”，并提供“立即启动Steam”按钮。

## 2. 系统架构（模块化设计）

三层架构（不变），这里补充 **各层应该实现的接口** 和 **职责清单**。

### 表示层（UI）
- **职责**：只做展示和用户交互，不包含业务逻辑。
- **必须提供的控件**：
  - 一个“开始加速安装”主按钮。
  - 一个进度条（0%~100%）。
  - 一个多行日志文本框（只读，带滚动）。
  - 一个“浏览...”按钮选择安装目录（默认为系统默认路径）。
  - 一个“保留加速（不恢复Hosts）”复选框。
- **数据绑定**：
  - 绑定到一个 ViewModel，包含 `ProgressValue`、`StatusText`、`IsRunning` 等属性。
  - 所有长时间操作必须异步，UI 线程不阻塞。

### 业务逻辑层
- **核心服务接口**：
  ```csharp
  interface ISteamInstallerService
  {
      Task<InstallResult> InstallAsync(InstallOptions options, IProgress<InstallProgress> progress, CancellationToken ct);
  }
  ```
- **InstallOptions** 包括：目标安装路径、是否保留加速、是否静默、日志路径等。
- **InstallProgress** 包括：总体进度百分比、当前步骤名称、详细消息。

### 数据服务层
- **Hosts 管理器接口**：
  ```csharp
  interface IHostsManager
  {
      Task BackupAsync();
      Task WriteEntriesAsync(IEnumerable<HostEntry> entries);
      Task RestoreAsync();
  }
  ```
- **加速节点提供者接口**：
  ```csharp
  interface IAccelNodeProvider
  {
      Task<IList<HostEntry>> FetchAsync();
  }
  ```

## 3. 核心模块详解

### 3.1 网络加速模块（Hosts管理路径 – 详细步骤）

#### 3.1.1 内置节点提供者
1. **准备一份默认的 IP 列表**（硬编码）
   - 例如：`23.36.68.116 steamcommunity.com` （仅为示例，实际需定期验证）
   - 列表应至少包含 4 个关键域名。
2. **实现 `IAccelNodeProvider`**
   - `FetchAsync()` 方法：
     1. 尝试从远程 URL（如 `https://your-server.com/steam-hosts.json`）拉取最新 JSON。
     2. 若远程拉取成功且格式正确，使用远程数据；否则回退到硬编码列表。
     3. 超时时间 3 秒。
   - 数据格式示例：
     ```json
     {
       "entries": [
         {"host": "steamcommunity.com", "ip": "1.2.3.4"},
         ...
       ]
     }
     ```

#### 3.1.2 Hosts 管理器具体动作
- **BackupAsync()**：
  1. 确定 hosts 文件路径：`Environment.GetFolderPath(Environment.SpecialFolder.System) + @"\drivers\etc\hosts"`。
  2. 若文件不存在，创建一个空的备份标记。
  3. 将 hosts 文件内容复制到 `Path.GetTempFileName()` 并记录备份路径。
- **WriteEntriesAsync()**：
  1. 读取当前 hosts 全部行。
  2. 对每个要添加的条目，检查该域名是否已存在于 hosts 中（忽略注释行）。
  3. 如果已存在但 IP 不同，则替换该行；如果不存在，追加到末尾。
  4. 写入文件（使用 `File.WriteAllLines`），确保编码为 UTF-8 without BOM。
  5. 调用 `Process.Start("ipconfig", "/flushdns")` 并等待退出。
- **RestoreAsync()**：
  1. 从备份路径恢复 hosts 文件。
  2. 如果备份不存在（首次未修改），则清除我们新增的条目（通过保留原始读取的副本实现）。
  3. 再次刷新 DNS。

### 3.2 静默安装模块

#### 安装步骤细化
1. **确定安装包路径**：临时目录下的 `SteamSetup.exe`。
2. **构建参数**：
   - 基本：`/S`
   - 若用户指定了目录 `D:\Games\Steam`，则参数为 `$"/S /D={targetDir}"`，**注意目录后不加引号**，即使路径含空格。
   - 检查参数长度不超过 Windows 限制（通常 32767 字符，但实际远小于此，安全）。
3. **启动进程**：
   ```csharp
   var psi = new ProcessStartInfo
   {
       FileName = setupPath,
       Arguments = arguments,
       UseShellExecute = false,
       CreateNoWindow = true,
       WindowStyle = ProcessWindowStyle.Hidden
   };
   var process = Process.Start(psi);
   ```
4. **等待安装完成**：
   - 由于安装器会自解压并启动 `Steam.exe`，简单等待安装进程退出不够，应同时启动进程监控。

### 3.3 UI / UX 辅助功能

- **日志输出规范**：
   - 时间戳 + 级别（INFO/WARN/ERROR） + 消息。
   - 使用 Serilog 同时输出到 UI 文本框和文件。
- **取消操作**：
   - 提供“取消”按钮，绑定 `CancellationTokenSource.Cancel()`。
   - 下载任务必须响应取消请求。

## 4. 安装执行模块深度设计

### 4.1 状态机详细动作（每个状态的“该做什么”）

#### 状态 `Idle`
- 初始化日志系统（Serilog 配置）。
- 新建 `CancellationTokenSource`。
- UI 按钮启用，状态文本“就绪”。
- 加载用户配置（上一次选择的安装路径、是否保留加速等）。
- 进入 `PreCheck`。

#### 状态 `PreCheck`
1. **管理员检查**：
   - 代码：`new WindowsPrincipal(WindowsIdentity.GetCurrent()).IsInRole(WindowsBuiltInRole.Administrator)`。
   - 如果不是，弹出提示框，确认后以管理员重启自身。
2. **磁盘空间检查**：
   - 获取默认安装路径或用户预选路径的根驱动器。
   - 可用空间 < 2 GB → 提示“磁盘空间不足”，转移到 `Error` 状态。
3. **网络检查**：
   - `HttpClient.GetAsync("https://store.steampowered.com")`，超时5秒。
   - 失败 → 提示“无法连接Steam服务器”，转到 `Error`。
4. **已有 Steam 检查**：
   - 若有 steam 进程，询问后结束进程，否则转到 `Error`（用户拒绝）。
5. 全部通过 → 进入 `FetchAccelNodes`。

#### 状态 `FetchAccelNodes`
1. 调用 `IAccelNodeProvider.FetchAsync()` 获取节点列表。
2. 若列表为空 → 记录警告，仍可继续（不使用加速），进入 `EnableHostsAccel` 但跳过实际写入。
3. 保存节点列表到上下文。

#### 状态 `EnableHostsAccel`
1. 调用 `IHostsManager.BackupAsync()` 备份原 hosts。
2. 调用 `IHostsManager.WriteEntriesAsync(nodes)` 写入映射。
3. 如果写入失败（权限不足等），记录错误，进入 `Error`。
4. 成功 → `DownloadSteamSetup`。

#### 状态 `DownloadSteamSetup`
1. 确定下载 URL（官方 CDN）。
2. 创建 `HttpClient`，设置超时 30 分钟，自动解压缩关闭。
3. 检查临时文件是否存在，若存在获取长度，设置 `Range: bytes=已下载-`。
4. 发送 GET 请求，如果服务器返回 206 则续传；200 则从头下载。
5. 边读边写文件流，报告进度（百分比、速度）。
6. 支持取消：检查 `CancellationToken`。
7. 下载完成，记录最终大小。

#### 状态 `VerifyDownload`
1. 获取服务器返回的 Content-Length（若有），与本地文件长度比较。
2. 若不等 → 删除文件，重新进入下载状态（最多重试3次）。
3. 可选：计算文件 MD5/SHA1 与官方公布值比较（若可得）。
4. 通过 → `RunSilentInstall`。

#### 状态 `RunSilentInstall`
1. 构造 `ProcessStartInfo`，文件名=下载的 setup，参数如前述。
2. 启动进程。
3. 记录安装进程 ID。
4. 立即进入 `MonitorInstall`。

#### 状态 `MonitorInstall`
**目标**：检测 steam.exe 出现或安装进程退出，判定安装成功与否。
- **动作**：
  1. 启动一个循环，每 2 秒检查一次。
  2. 检查 `steam.exe` 进程是否存在（`Process.GetProcessesByName("steam")`）。
  3. 若存在：安装可能成功，等待 3 秒让注册表写入，然后转到 `VerifyInstall`。
  4. 若安装进程已退出但 `steam.exe` 未出现：判定安装失败，转到 `Error`。
  5. 设置超时 5 分钟，超时则转到 `Error`。
- 在循环中可更新 UI：“正在等待Steam启动...”

#### 状态 `VerifyInstall`
1. 读取注册表键 `HKLM\SOFTWARE\WOW6432Node\Valve\Steam\InstallPath`，获取安装目录。
2. 检查该目录下 `steam.exe` 是否存在。
3. 若注册表没有，尝试在常见默认路径查找：`C:\Program Files (x86)\Steam\steam.exe`。
4. 若都不存在 → 进入 `Error`。
5. 成功 → `CleanupAndFinish`。

#### 状态 `CleanupAndFinish`
1. 若用户未选择“保留加速”，调用 `IHostsManager.RestoreAsync()`。
2. 删除临时 SteamSetup.exe。
3. 删除 Hosts 备份文件。
4. 释放资源。
5. 设置最终状态为“安装完成”，UI 显示安装路径和“启动Steam”按钮。
6. 转到 `Finished`。

#### 状态 `Error`
1. 记录详细错误信息和堆栈到日志。
2. 尝试恢复 Hosts（无论用户设置，安全优先）。
3. 清理临时文件。
4. UI 显示错误消息，并提供“查看日志”按钮。
5. 转到 `Finished`。

#### 状态 `Finished`
- 释放 CancellationTokenSource。
- 禁用取消按钮，主按钮变为“重新开始”。
- 不自动退出。

### 4.2 任务管道备选设计详解

如果使用任务管道，每个 `IInstallTask` 实现需按以下规范：

1. **PreCheckTask**：
   - ExecuteAsync 内串行执行 3.1 节的所有检查步骤。
   - 任何检查失败返回 false，上下文 `ErrorMessage` 设置。

2. **FetchAccelNodesTask**：
   - 调用加速节点服务，结果存入 `context.Nodes`。

3. **BackupHostsTask**：
   - 调用 `hostsManager.BackupAsync()`。

4. **WriteHostsTask**：
   - 调用 `hostsManager.WriteEntriesAsync(context.Nodes)`。

5. **DownloadSteamSetupTask**：
   - 实现下载逻辑，带重试次数（如3次）。
   - 进度汇报到 `context.Progress`。

6. **RunSilentInstallTask**：
   - 启动安装进程，并将进程对象存入 `context.InstallProcess`。

7. **WaitForSteamProcessTask**：
   - 循环监控，设置超时，成功或失败返回 bool。

8. **VerifyInstallationTask**：
   - 执行注册表/文件验证，设置 `context.InstallPath`。

9. **RestoreHostsTask**：
   - 根据 `context.Options.KeepAcceleration` 决定是否调用恢复。

10. **CleanupTempTask**：
    - 删除临时文件。

每个任务记录开始和结束日志，便于追踪。

## 5. 安装路径的监控与设置（补充详细行动）

### 5.1 指定安装路径
- **UI端**：用户点击“浏览”按钮，调用 `FolderBrowserDialog` 或 `OpenFolderDialog`，返回路径。
- **构造参数**：
  ```csharp
  string args = "/S";
  if (!string.IsNullOrEmpty(targetDir))
  {
      args += $" /D={targetDir}"; // 无引号，即使路径有空格
  }
  ```
- **常见问题处理**：如果用户目录名含尾部空格，需 `TrimEnd()`，否则可能安装失败。

### 5.2 安装后检测路径（更完善）
- **注册表读取**：考虑32位应用在64位系统上的重定向，务必显式指定 `RegistryView.Registry64`。
- **代码**：
  ```csharp
  using (var baseKey = RegistryKey.OpenBaseKey(RegistryHive.LocalMachine, RegistryView.Registry64))
  using (var steamKey = baseKey.OpenSubKey(@"SOFTWARE\WOW6432Node\Valve\Steam"))
  {
      return steamKey?.GetValue("InstallPath") as string;
  }
  ```
- **备用检测**：遍历驱动器根目录搜索 `steam.exe`（通过 Everything SDK 或简单递归，但注意性能，最多搜索两层）。

### 5.3 高级监测（实时捕获注册表变化）
- 使用 `RegistryMonitor` 或 WMI 事件监听注册表 `Valve\Steam` 项下的值变化，但这非必须，仅用于超精确判断安装完成时刻。常规轮询已足够。

## 7. 开发落地步骤（MVP → 迭代）—— 每步具体做什么

### 第一阶段（MVP）—— 核心流程打通

#### 步骤 0：项目初始化
- 创建 WPF 或 Avalonia 项目（.NET 8）。
- 添加文件夹结构：`Models`, `Services`, `ViewModels`, `Views`。
- 安装 Serilog 包，配置 `App.xaml.cs` 中初始化日志。

#### 步骤 1：实现最简单的 UI
- MainWindow 添加：
  - “开始加速安装”按钮，绑定命令 `StartCommand`。
  - ProgressBar，`Value` 绑定 `ProgressValue`。
  - TextBox（多行，只读），绑定 `StatusLog` 并自动滚动。
- 对应的 MainViewModel 实现 `INotifyPropertyChanged`。

#### 步骤 2：PreCheck 模块
- 创建 `PreCheckService`，实现：
  - `CheckAdmin()`：返回 bool，若 false 提示并尝试重启。
  - `CheckDiskSpace(string path, long requiredBytes)`。
  - `CheckNetwork()`：简单的 `Ping` 或 HTTP GET。
  - `KillExistingSteam()`：弹出 MessageBox 确认后 kill。
- 在 ViewModel 的 StartCommand 开始时调用，结果记录日志。

#### 步骤 3：Hosts 加速
- 创建 `HostsFileManager` 类，实现 `Backup`, `Write`, `Restore`。
- 硬编码几个加速 IP（如 `steamcommunity.com -> 23.36.68.116`）。
- 注意：需要管理员权限，在写入时 `try-catch UnauthorizedAccessException`。

#### 步骤 4：下载 SteamSetup.exe
- 创建 `DownloadService`：
  - `HttpClient` 带进度报告的扩展方法。
  - 下载到 `Path.GetTempPath() + "SteamSetup.exe"`。
  - 报告进度更新 ViewModel 的 ProgressValue。
- 下载完成校验文件存在且大小 > 1MB（不严格）。

#### 步骤 5：静默安装
- 在 ViewModel 中调用 `Process.Start`，不指定路径，让 Steam 安装到默认位置。
- 参数 `"/S"`。

#### 步骤 6：进程监控
- 使用 `Task.Run` 启动循环，每秒检查 `Process.GetProcessesByName("steam")`。
- 若发现 steam.exe，继续等待 3 秒，然后认为安装完成。
- 更新 UI 状态。

#### 步骤 7：安装验证
- 尝试读取注册表，若读取失败，检查默认 `Program Files (x86)\Steam\steam.exe`。
- 若存在，标记成功。

#### 步骤 8：清理
- 删除临时安装包。
- 如果用户未勾选保留加速，恢复 Hosts。
- 显示完成消息。

### 第二阶段（增强稳定性）—— 每一步做什么

#### 步骤 9：断点续传
- 在 DownloadService 中检测临时文件是否存在，设置 `Range` 头。
- 合并文件流写入模式为追加。
- 添加重试逻辑：网络中断时自动重试 3 次，每次间隔 2 秒。

#### 步骤 10：远程加速 IP 更新
- 搭建或使用公开的加速 IP 列表（如 JSON 托管在 GitHub Pages）。
- 在 `AccelNodeProvider` 中实现 HttpClient 拉取，解析 JSON。
- 缓存到本地文件，有效期 24 小时。

#### 步骤 11：用户选择安装目录
- View 中添加“浏览”按钮和路径文本框。
- 将选择的路径传入安装参数，注意 `/D` 参数格式。
- 增加路径合法性检查：不允许根目录，路径长度 < 200，等等。

#### 步骤 12：详细日志记录
- 在关键步骤前后记录 `Log.Information`。
- 异常捕获记录 `Log.Error`。
- 当日志文件超过 10MB 时自动分割。

#### 步骤 13：错误回滚机制
- 在任何步骤失败时，统一进入错误处理方法，确保：
  - 恢复 Hosts（调用 Restore）。
  - 删除半成品文件。
  - 显示具体错误和建议。

### 带断点续传的下载方法
```csharp
public async Task DownloadWithResumeAsync(string url, string destPath, IProgress<double> progress, CancellationToken ct)
{
    long existingLength = 0;
    if (File.Exists(destPath))
    {
        existingLength = new FileInfo(destPath).Length;
    }

    using var client = new HttpClient { Timeout = TimeSpan.FromMinutes(30) };
    var request = new HttpRequestMessage(HttpMethod.Get, url);
    if (existingLength > 0)
    {
        request.Headers.Range = new System.Net.Http.Headers.RangeHeaderValue(existingLength, null);
    }

    var response = await client.SendAsync(request, HttpCompletionOption.ResponseHeadersRead, ct);
    response.EnsureSuccessStatusCode();

    long totalLength = existingLength;
    if (response.Content.Headers.ContentLength.HasValue)
    {
        totalLength += response.Content.Headers.ContentLength.Value;
    }

    using var fileStream = new FileStream(destPath, FileMode.Append, FileAccess.Write, FileShare.None);
    using var networkStream = await response.Content.ReadAsStreamAsync();

    byte[] buffer = new byte[8192];
    long totalRead = existingLength;
    int read;
    while ((read = await networkStream.ReadAsync(buffer, 0, buffer.Length, ct)) > 0)
    {
        await fileStream.WriteAsync(buffer, 0, read, ct);
        totalRead += read;
        progress.Report((double)totalRead / totalLength * 100);
    }
}
```

### 读写 Hosts 文件（防权限问题）
```csharp
public async Task WriteEntriesAsync(IEnumerable<HostEntry> entries)
{
    string hostsPath = Path.Combine(Environment.GetFolderPath(Environment.SpecialFolder.System), "drivers/etc/hosts");
    string tempPath = Path.GetTempFileName();
    bool modified = false;

    // 读取原文件并处理
    var lines = (await File.ReadAllLinesAsync(hostsPath)).ToList();
    foreach (var entry in entries)
    {
        string pattern = $"{entry.Ip} {entry.Host}";
        int index = lines.FindIndex(l => l.TrimEnd().EndsWith(entry.Host, StringComparison.OrdinalIgnoreCase) && !l.TrimStart().StartsWith("#"));
        if (index >= 0)
        {
            lines[index] = pattern;
            modified = true;
        }
        else
        {
            lines.Add(pattern);
            modified = true;
        }
    }

    if (modified)
    {
        await File.WriteAllLinesAsync(tempPath, lines, Encoding.UTF8);
        File.Copy(tempPath, hostsPath, overwrite: true);
        File.Delete(tempPath);
        // 刷新 DNS
        Process.Start("ipconfig", "/flushdns")?.WaitForExit(2000);
    }
}
```
